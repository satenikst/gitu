Պահեստի կարգավորումներ

git init

git init հրամանը ստեղծում է նոր Git շտեմարան։ Սա օգտագործվում է արդեն գոյություն ունեցող Git շտեմարանը փոխելու կամ նոր Git շտեմարան ստեղծելու համար։ Այսպիսով, սա առաջին հրամանն է, որը դու կօգտագործես նոր նախագիծ ստեղծելիս։ git init-ը նախագծի հիմքում ստեղծում է .git ենթացուցակ, որն իր մեջ պարունակում է բոլոր կարևոր տվյալները։

կիրառում
git init

git clone

git clone հրամանը կրկնօրինակում է (copy) արդեն գոյություն ունեցող git շտեմարանը։ clone-ը ավտոմատ կերպով ստեղծում է հեռակա կապ դեպի օրիգինալ շտեմարան։ Սրա միջոցով հնարավոր է դառնում հեշտությամբ համագործակցել կենտրոնական շտեմարանի հետ։

կիրառում
git clone

Շտեմարանի կրկնօրինակը գտնվում է քո լոկալ PC-ի մեջ։ Օրիգինալ շտեմարանը կարող է տեղակայված լինել լոկալ ֆայլային համակարգի վրա կամ հեռակա, որը հասանելի է HTTP-ի կամ SSH-ի միջոցով։

git config

git config հրամանը քեզ թույլ է տալիս ձևակերպել Git-ի տեղակայումը (installation) command line-ից։

կիրառում
git config user.name <.name.>

Սահմանվում է հեղինակի անունը, որը կօգտագործվի տվյալ շտեմարանում բոլոր փոփոխությունների համար։ Նաև պետք է օգտագործել --global-ը տվյալ օգտագործողի համար փոփոխությունների կարգավորումներ կատարելու դեպքում։

git config --global user.name <․name․>

Սահմանվում է հեղինակի էլեկտրոնային փոստի հասցեն, որպեսզի բոլոր փոփոխությունները կատարվեն ճիշտ օգտագործողի կողմից։

git config --global user.email <․email․>

Git հրամանի համար ստեղծվում է shortcut

git config --global alias.<.alias-name.> <.git-command.>

Սահմանում է տեքստային խմբագրիչ (text editor)։ <․editor․>-ը հրաման է, որը գործարկում է (run) քեզ անհրաժեշտ խմբագրիչը։

git config --system core.editor <․editor․>

Խմբագրիչի մեջ բացում է գլոբալ կարգավորումների ֆայլը, արդեն իսկ խմբագրելու համար։

git config --global --edit
