Փոփոխությունների պահպանում

git add

git add հրամանի միջոցով ավելացվում են կատարված փոփոխությունները։ Սա Git-ին հաղորդում է, որ դու ցանկանում ես ավելացնել ֆայլի թարմացումները հաջորդ commit-ում։ Սակայն, git add-ը իրականում ոչ մի ազդեցություն չունի շտեմարանի վրա։ Փոփոխություններ չեն կատարվում այնքան ժամանակ, մինչև դու գործի չես դնում git commit հրամանը։

Այս հրամանների հետ մեկտեղ դու կարիք ունես գործարկելու git status հրամանը։ Սրա միջոցով դու կարող ես տեսնել քո աշխատանքային միջավայրի վիճակը։

կիրառում
git add 'file'

git commit

git commit հրամանի միջոցով նախագծում ֆիքսվում են տեղի ունեցած փոփոխությունները։ Git-ը երբեք փոփոխություն չի կատարի մինչև դու չօգտագործես այս հրամանը։ git add-ը և git commit-ը Git-ի կարևորագույն հրամաններից են համարվում։

կիրառում
git commit

git commit հրամանը նույնպես հնարավորություն է տալիս գրել հաղորդագրություն տվյալ փոփոխությանը վերաբերվող։

git commit -m 'message'
