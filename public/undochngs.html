ՓՈՓՈԽՈՒԹՅՈՒՆՆԵՐԻ ՉԵՂԱՐԿՈՒՄ

git checkout

git checkout հրամանը կատարում է հետևյալ գործողությունները՝

  1. ստուգում է ֆայլերը,
  2. ստուգում է commit-ները,
  3. ստուգում է ճյուղերը

Այս մոդուլում մենք գործ կունենանք միայն առաջին երկուսի հետ։ Ստուգելով commit-ը ամբողջ աշխատանքային ցուցակը համապատասխանեցվում է այդ commit-ին։ Սա կարող է օգտագործվել քո նախագծի հին վիճակը տեսնելու համար, չփոխելով ընթացիկ վիճակը։ Ֆայլը ստուգելով դու կարող ես տեսնել ֆայլի հին տարբերակը, նույնպես առանց ընթացիկ վիճակի փոփոխության։

կիրառում
git checkout master

Սա վերադարձնում է master ճյուղը։

git checkout 'commit' 'file'

Սա ստուգում է ֆայլի նախկին տարբերակը։

git checkout 'commit'

Սա թարմացնում է աշխատանքային ցուցակի բոլոր ֆայլերը՝ տվյալ commit-ին համապատասխան։

git revert

git revert հրամանը ձևափոխում է commit-ները։ Բայց, նախագծի պատմությունից commit-ները ջնջելու փոխարեն չեղարկում է եղած փոփոխությունները և ավելացնում է նոր commit: Սա պաշտպանում է Git-ին պատմության կորստից, որը շատ կարևոր է ձեր նախագծի պատմության ամբողջականության և հուսալի աշխատանքի համար։

կիրառում
git revert 'commit'

Սա գեներացնում է նոր commit, որը չեղարկում է 'commit'-ում ներկայացված բոլոր փոփոխությունները։

git reset

Եթե git revert-ը հուսալի ճանապարհ է փոփոխությունների համար, ապա ես դա չեմ կարող ասել git reset հրամանի մասին։Երբ դու չեղարկում ես git reset-ով, ապա այստեղ չի փոփոխվում օրիգինալի կրկնօրինակը, այլ այն փոփոխվում է մշտապես։ Այս մեթոդը վտանգավոր է և համարվում է Git-ի այն հրամաններից մեկը, որոնք ունակ են կործանելու քո աշխատանքը։

կիրառում
git reset 'file'

Սա ջնջում է տվյալ ֆայլը նախագծից, բայց աշխատանքային միջավայրը թողնում է անփոփոխ։

git log -n 'limit'

Սահմանափակում է commit-ների քանակը ըստ՝ 'limit'-ի։

git clean

git log հրամանը ցուցադրում է տեղի ունեցած բոլոր commit-ները։ Դա քեզ թույլ է տալիս թերթել նախագծի պատմության էջերը, զտել (filter) այն, և փնտրել հատուկ փոփոխություններ։

կիրառում
git log

Սրանով ցուցադրվում է գոյություն ունեցող commit-ների պատմությունը։

git log -n 'limit'

Սահմանափակում է commit-ների քանակը ըստ՝ 'limit'-ի։
