git commit --amend

git commit --amend-ի միջոցով ֆիքսվում են վերջին commit-ները։ Սա հնարավորություն է տալիս համախմբել կատարված փոփոխությունները նախորդ commit-ի միջոցով, այն որպես ամբողջովին նոր պատկերով ֆիքսելու փոխարեն։ Այն նույնպես կարող է օգտագործվել նախորդ commit-ի նամակը խմբագրելու համար, առանց փոփոխելու նրա պատկերը։

git commit --amend

git rebase

rebase գործողությունը համարվում է տվյալ ճյուղի (branch) անցումը դեպի ֆիսքման նոր հիմք։ Ըստ տեսության rebase գործողությունը կարողանում է ճյուղը տեղափոխել մեկ commit-ից մեկ այլ commit: Բայց իրականում դա կատարվում է այլ կերպ։ Git-ը դա իրականացնում է նոր commit-ներ ստեղծելով և դրանց տալով հատուկ հիմք։ Դա վերափոխում է քո նախագծի պատմությունը։ Թեև ճյուղի տեսակը մնում է նույնը, բայց դա պարունակում է ամբողջովին նոր commit-ներ։

git log

Սրանով ցուցադրվում է գոյություն ունեցող commit-ների պատմությունը։

git rebase 'base'

Վերահիմնավորում է տվյալ ճյուղը, որը կարող է լինել ցանկացած տեսակի commit-ի մեջբերում:

git rebase -i

git log հրամանը ցուցադրում է տեղի ունեցած բոլոր commit-ները։ Դա քեզ թույլ է տալիս թերթել նախագծի պատմության էջերը, զտել (filter) այն, և փնտրել հատուկ փոփոխություններ։
